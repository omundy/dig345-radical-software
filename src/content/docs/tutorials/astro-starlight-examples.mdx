## Asides 

[Asides](https://starlight.astro.build/guides/authoring-content/#asides) can be used in `.md` and `.mdx` files



Starlight's custom aside syntax looks like this: 

```
:::note
A note aside
:::
```

... to create this:


:::note
A note aside
:::

:::caution
A caution aside
:::

:::danger
A danger aside
:::

## Components


### Built-in Components

Import a component into an `.mdx` file in Starlight like this: 

```mdx
import { Card } from '@astrojs/starlight/components';

<Card title="Check this out">Interesting content you want to highlight.</Card> 
```

... which results in this:

import { Card } from '@astrojs/starlight/components';

<Card title="Check this out">Interesting content you want to highlight.</Card> 



### Other components

import { Icon } from '@astrojs/starlight/components';

<Icon name="open-book" class="inline-icon" />  Learn more about [icons](https://starlight.astro.build/reference/icons/#all-icons)

## About the Diátaxis framework

- A [Tutorial](https://diataxis.fr/tutorials/) is a practical activity, in which the student learns by doing something meaningful, towards some achievable goal.
- [How-to guides](https://diataxis.fr/how-to-guides/) lead a user through a specific task they want to accomplish, often with a sequence of steps. Writing a good guide requires thinking about what your users are trying to do.
- [Reference pages](https://diataxis.fr/reference/) are ideal for outlining how things work in terse and clear terms. Less concerned with telling a story or addressing a specific use case, they should give a comprehensive outline of what you're documenting.
- An [Explanation](https://diataxis.fr/explanation/) provides context and background.
