#### Create an Astro blog site 

Follow [Astro's Build a Blog tutorial](https://docs.astro.build/en/tutorial/0-introduction/) to create a modular website using component-based design. Copy and paste these instructions into `devlog-2-3` in your DevLog. Then publish it and turn it in by posting the direct link to this DevLog on Moodle.


1. Create a new Astro site and publish it on Netlify [<Icon name="document" class="inline-icon" />](https://docs.astro.build/en/tutorial/1-setup/). Add the formatted (backticks) command ‚úèÔ∏è here 
1. Create pages (`.astro` files) and blog posts (`.md` files) using frontmatter and file-based routing [<Icon name="document" class="inline-icon" />](https://docs.astro.build/en/tutorial/2-pages/). 
1. Use Astro [components](https://docs.astro.build/en/basics/astro-components/) (also `.astro` files!) to reuse common code across your website [<Icon name="document" class="inline-icon" />](https://docs.astro.build/en/tutorial/3-components/). 
1. Create reusable layout components using `<slot />` and `Astro.props` to inject child content [<Icon name="document" class="inline-icon" />](https://docs.astro.build/en/tutorial/4-layouts/). 
1. Create layout components for markdown files
1. Display a dynamically generated list of posts on your Blog page [<Icon name="document" class="inline-icon" />](https://docs.astro.build/en/tutorial/5-astro-api/). 
1. Use Dynamic page routing with [`getStaticPaths()`](https://docs.astro.build/en/tutorial/5-astro-api/2/) to specify and store static routes during build, and pass each page its own props.
1. Install an Astro Preact integration. Add a components in their own language. Use a client: directive for hydration on my UI framework component. https://docs.astro.build/en/tutorial/6-islands/1/
1. Compare your finished blog site to the [tutorial example](https://github.com/withastro/blog-tutorial-demo/tree/complete) (I used Svelte, but it is nearly the same effect)
1. Modify your site design and content as if you were transitioning to Astro.js from an existing Wordpress site/blog. Meaning, move content into your site.
